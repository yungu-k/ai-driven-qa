> - **그래서**: 스크린샷 재촬영 + Issues 뷰 반영 + (배포됐다면) 처리 기록 기능 추가.

> 확인된 사실: `monitor-downtime.png`는 **정상 렌더**(막대·추이선 정상). 깨진 건 `monitor-analysis.png` 하나. `monitor-status.png`는 차트가 없어 위험 낮음(확인만).

## 📋 개발 핸드오프

### M1. 🔴 분석 뷰 스크린샷 재촬영 (차트 비어 있음)

- **무엇을**: `public/images/monitor-analysis.png`를 **차트가 다 그려진 뒤** 다시 캡처해 교체한다.
- **원인**: 차트가 브라우저에서 비동기로 그려지는데, 렌더 완료 전에 스크린샷이 찍혔다. 축·범례·KPI 카드만 남고 **막대·추이선이 전부 비어 있음**.
- **수용 기준**:
  - [ ] 교체된 이미지에서 **Top Alarm Codes 막대**, **By Machine 막대**, **Daily Trend 선**이 **실제로 그려져** 보인다.
  - [ ] 캡션("Action Required vs Guide Exists 비교와 일별 추이를 본다")이 이미지와 실제로 일치한다.
  - [ ] 다른 두 이미지(status·downtime)와 톤·해상도·다크테마가 일관.
  - [ ] 캡처 시 **차트 렌더 완료를 기다린 뒤** 찍는다(고정 대기 또는 렌더 완료 신호 확인).
- **영향 범위**: `public/images/monitor-analysis.png` 교체(파일명 유지 시 본문 수정 불필요).
- **제약**: 민감정보 노출 금지(고객사명·실제 티켓·IP 등 화면에 안 잡히게). 기존 이미지와 동일 파일명 권장.

### M2. 🔴 "세 개의 화면" → 실제 화면 수 정정 + Issues 뷰 반영

- **무엇을**: 본문이 "세 개의 화면으로 나눴다"(상태/분석/가용성)인데, 실제 대시보드와 스크린샷 메뉴는 **Status · Analysis · Issues · Downtime 4개**다. **Issues 뷰가 글에 없다.**
- **수용 기준**:
  - [ ] "세 개의 화면" 서술이 **실제 개수로 정정**된다.
  - [ ] **Issues 뷰 설명 한 항목 추가** — 다른 뷰와 같은 형식(한 줄 요지 + 무엇을 보는 화면인지). 배포일 기준으로 어떤 에러가 늘고 줄었는지 추적하는 화면이라는 취지.
  - [ ] 본문 뒤쪽 "분류 기준 한 곳" 문장의 "**상태·분석·가용성 세 화면에 동시 반영**"도 화면 수에 맞게 정정(코드 주석 포함).
  - [ ] (선택) Issues 뷰 스크린샷 1장 추가 — 넣는다면 M1과 같은 기준(차트 렌더 후 캡처).
- **영향 범위**: `src/content/posts/realtime-kiosk-monitoring.md` (본문 목록 + 코드 발췌 주석).
- **제약**: 평서체·톤 유지. 없는 기능 지어내지 말 것 — **실제 Issues 뷰가 하는 일만** 서술.

### M3. 🟡 처리 기록(담당자·사유) 반영 — **배포 확인 후에만**

- **무엇을**: 최근 대시보드에 "고장을 처리 완료로 끌 때 담당자·사유를 남기고 이력을 보존"하는 기능이 추가됐다. 이 글은 대시보드의 정본 글인데 이 기능이 없다.
- **선행 확인 (필수)**:
  - [ ] 이 기능이 **실제로 운영에 배포됐는지 먼저 확인**한다(작업 브랜치에만 있고 미배포일 수 있음). **미배포면 이 항목은 건너뛰고 그렇게 보고**한다.
- **수용 기준(배포된 경우에만)**:
  - [ ] 상태 월보드 항목이나 별도 한 줄로 **처리 기록(담당자·사유 + 이력)** 서술 추가.
  - [ ] 과장 없이 — 실제 동작만.
- **영향 범위**: `realtime-kiosk-monitoring.md`.
- **참고**: 이 기능은 `ai-developer-teammate` 글에서 개발자 팀원의 파일럿 사례로 이미 언급됨 → 두 글의 정합이 맞아진다.

### M4. 🟢 코드 발췌의 알람 코드 (확인만)

- **무엇을**: 본문 `app/config.py` 발췌의 `WARNING_CODES` 예시 코드값이 실제 설정과 다르다. **의도적 각색이면 그대로 두고**, 실수라면 실제 값 계열로 맞춘다.
- **수용 기준**: [ ] 각색 여부 판단 후 유지 또는 정정. 사내 실제 코드 노출이 부담이면 **각색 유지가 맞다**(그 경우 변경 없음).
