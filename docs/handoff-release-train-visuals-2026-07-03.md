## 📋 개발 핸드오프

### T1. 스프린트 배분 — 타임라인 시각화 + 표 정리

**문제**: 현재 표의 `└ 신규/변경` · `└ 회귀` 열이 `QA` 열과 **동등한 열**로 보인다(마크다운 표는 들여쓰기가 안 먹음). 포함 관계가 병렬로 읽혀 합계를 오해할 수 있다.

- **무엇을**: (1) 스프린트 흐름을 **가로 타임라인 막대**로 시각화하고, (2) 표는 **열을 줄여** 포함 관계를 셀 안에서 표현한다.

- **타임라인 사양 (기획 결정)**:
  - 두 줄: **2주(10일)**, **3주(15일)**. 각 줄의 가로 전체 = 그 스프린트의 총 작업일(비율 정확히).
  - 세그먼트: **개발**(6일 / 9일) → **QA**(4일 / 6일). **QA 구간 안을 다시 신규·변경(3/4일)과 회귀(1/2일)로 하위 분할** — 이 중첩이 "QA 안에 두 단계가 있다"를 시각적으로 해결하는 핵심이다.
  - **Code Freeze(개발 마감) 지점**을 개발↔QA 경계에 **마커+라벨**로 표시. 본문의 "Code Freeze = QA 착수일"이 그림으로 증명된다.
  - **직접 라벨**: 각 세그먼트에 일수 표기(6일 / 3일 / 1일 …). 좁아서 안 들어가면 범례로.
  - **범례** 필수(개발 / QA-신규·변경 / QA-회귀 3계열).
  - 회귀 구간이 **맨 마지막**에 오는 것이 눈에 보이게 — 본문 "회귀는 마지막에" 주장과 그림이 일치해야 한다.

- **표 정리 사양**:
  - 열을 **6개 → 4개**로: `스프린트 | 총 작업일 | 개발 | QA`
  - QA 셀 안에서 내역을 표기: 예) **`4일 (신규·변경 3 + 회귀 1)`**, **`6일 (신규·변경 4 + 회귀 2)`**
  - `└` 기호는 제거(마크다운 표에서 위계를 못 만든다).
  - **수치는 변경 없음**(2주 10=6+4, QA 4=3+1 / 3주 15=9+6, QA 6=4+2).

- **수용 기준**:
  - [ ] 타임라인에서 **개발→QA 순서, QA 안의 신규→회귀 순서**가 한눈에 보인다.
  - [ ] **Code Freeze 지점**이 표시된다.
  - [ ] 세그먼트 폭이 실제 일수 **비율과 일치**한다(2주: 60/40, 그중 QA는 3:1).
  - [ ] 표가 4열로 정리되고 QA 내역이 **셀 안 괄호**로 들어간다. `└` 잔존 0.
  - [ ] **수치 무변경**.
  - [ ] 다크/라이트 정상.

### T2. 보드 운영 도식 (실제 스크린샷 없이)

- **무엇을**: "어떻게 굴리는가"를 보여주는 도식을 본문 "도구에 앉히기" 섹션에 넣는다. **실제 Jira 화면 캡처는 사용하지 않는다**(이슈키·고객사·제품 코드명·담당자 노출 위험).

- **필수 — 도식 ①: 상태 흐름**
  - `백로그 → 진행 중 → 개발 완료 → QA → 완료` 가로 흐름.
  - **강조점 2개가 그림에서 읽혀야 한다**:
    1. **QA가 옆 라인이 아니라 흐름 안에 있다** (본문 "QA는 별도 단계가 아니라 각 작업의 완료 조건").
    2. **'개발 완료'는 완료가 아니다** — 완료 이전 단계임이 시각적으로 구분(예: 완료만 색/테두리 강조, 개발 완료는 중립). 본문 함정 3번("개발 완료 = 끝 → 통계가 거짓말")과 연결.
  - 버그가 독립 티켓으로 원인 작업에 **연결**된다는 점을 곁가지로 표현해도 좋다(과하면 생략).

- **선택 — 도식 ②: 통합 보드 1개 + 팀별 뷰**
  - 통합 보드(정본)에서만 스프린트를 시작하고, 팀별 보드는 같은 티켓을 **보기만** 하는 뷰라는 구조.
  - 한 줄 규칙 **"스프린트는 한 곳에서만 시작한다"**를 그림 옆에 캡션으로.
  - 텍스트로도 전달되는 내용이라 **여유 없으면 생략 가능**.

- **수용 기준**:
  - [ ] 도식 ①이 있고, 위 강조점 2개가 그림만 봐도 읽힌다.
  - [ ] **실제 화면 캡처를 쓰지 않았다**(민감정보 0).
  - [ ] 도식 텍스트에 사내 용어·이슈키·고객사·제품 코드명이 없다(일반 용어만).
  - [ ] 다크/라이트 정상.

## 구현 제약

- **정적 유지** — 새 JS 차트/다이어그램 라이브러리 도입 금지. **인라인 SVG 또는 CSS**로(DB 글의 CSS 막대와 같은 기법). 이미지 파일보다 인라인이 낫다(다크·라이트 자동 대응, 텍스트 검색 가능).
- **dataviz 방법 준수** — 타임라인은 3계열 categorical(개발 / QA-신규 / QA-회귀). **`scripts/validate_palette.js`로 light·dark 각각 실제 실행**해 통과한 색만 사용(눈으로 판단 금지). 기존 사이트 토큰 기반, 차트 전용 변수로 분리해 사이트 토큰은 무변경.
- **접근성** — 도식/타임라인에 `role="img"` + 수치를 서술한 `aria-label`. 표는 그대로 두어 정확값·대체 수단 역할.
- **본문 서술은 건드리지 않는다** — 표 자리와 도식 삽입 외 문장 수정 없음. 평서체·용어 병기·민감정보 0 유지.

- 빌드 green + **라이브 렌더 눈검증**(타임라인 비율·Code Freeze 마커·도식 가독성, 다크/라이트 둘 다).
- 팔레트 검증 스크립트 **실제 실행 결과 첨부**.
- 민감정보 재확인: 새로 넣은 도식 텍스트 포함 전체 재검색(이슈키·고객사·제품 코드명 0).
- 표 수치 무변경 확인(6/4/3/1, 9/6/4/2), `└` 잔존 0.

- 완료 보고는 돌린 확인 + 결과로.
