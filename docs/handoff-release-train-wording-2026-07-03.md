## 📋 개발 핸드오프

### W1. "회귀" → "Regression" (단계 이름 자리에 한해)

**근거**: 블로그가 이미 **단계·산출물 이름은 영문**으로 쓴다 — `regression-and-status-sync` **제목**("Regression TC 추출 …"), 여정기 **Phase 5 제목·표**("Regression + 상태 동기화"). 반면 "회귀 범위가 커진다" 같은 **현상 서술**은 한글이 자연스럽다. 이 글은 그 구분 없이 전부 "회귀"라 다른 글과 어긋난다.

- **무엇을**: 이 글에서 **QA 하위 단계의 "이름"으로 쓰이는 자리**를 `Regression`으로 바꾼다.
- **적용 범위 (이름 자리 = 바꾼다)**:
  - 타임라인 **세그먼트 라벨**("회귀 1일" → "Regression 1일")
  - 타임라인 **범례**("QA — 회귀" → "QA — Regression")
  - **표의 QA 셀**("신규·변경 3 + 회귀 1" → "신규·변경 3 + Regression 1")
  - 본문에서 그 **단계를 지칭**하는 문장("QA는 신규/변경 테스트를 먼저, **회귀**는 마지막에" → "…, **Regression**은 마지막에")
  - `figcaption`·`aria-label` 안의 해당 표현
- **바꾸지 않는 자리 (현상 서술 = 그대로)**:
  - "한 번에 다량을 배포하니 **회귀 범위**가 커지고" (배경 섹션) — 단계 이름이 아니라 현상 서술이라 한글이 자연스럽다.
  - 급행 트랙의 "영향 범위를 한정한 **회귀**를 돌리고" — 판단에 맡김(단계 이름에 가까우면 Regression, 서술이면 유지).
- **첫 등장 병기 1회**: 블로그 용어 정책대로 처음 나오는 곳에서 **`Regression(회귀) 테스트`** 로 한 번 풀고, 이후 `Regression`.
- **수용 기준**:
  - [ ] 타임라인 라벨·범례·표 셀·해당 본문 문장이 `Regression`으로 통일.
  - [ ] 첫 등장에 **우리말 병기 1회** 존재.
  - [ ] 배경 섹션의 "회귀 범위" 같은 **현상 서술은 유지**(무리한 전면 치환 금지).
  - [ ] **수치 무변경.**

### W2. ⚠ 레이아웃 — 글자가 잘리면 안 된다 (W1의 필수 동반 작업)

"Regression"은 "회귀"보다 훨씬 길다. 2주 스프린트의 Regression 세그먼트는 **폭이 10%**라 현재 방식(세그먼트 안 가운데 라벨)으로는 **넘치거나 잘린다**.

- **수용 기준**:
  - [ ] 어떤 화면 폭에서도 **라벨이 잘리거나 세그먼트를 넘치지 않는다**(모바일 포함).
  - [ ] **세그먼트 폭 비율은 그대로**(60/30/10, 60/26.667/13.333) — 글자 때문에 폭을 왜곡하지 말 것.
- **해결 방식은 재량** — 예: 라벨을 막대 **아래 행**으로 내리기 / 좁은 구간만 막대 **바깥에 표기** / 글자 크기·줄바꿈 조정 / 라벨을 짧게(`Regression 1일` → 범례로 위임하고 세그먼트엔 `1일`).

- 🔴 **단, 반드시 지킬 것 — 보조 인코딩을 잃지 말 것**:
  현재 이 타임라인은 **각 세그먼트가 자기 이름을 라벨로 달고 있어서** 색이 없어도 구분된다. 팔레트 검증에서 **다크 모드 tritan ΔE 6.3 = floor band**라, "직접 라벨 등 보조 인코딩이 있을 때만 적법"인 상태다.
  - [ ] 라벨을 세그먼트 밖으로 빼더라도 **어느 구간이 무엇인지 색 없이 식별 가능**해야 한다(인접 라벨·지시선·순서가 명확한 범례 등).
  - [ ] 라벨을 전부 없애고 **색만으로 구분하게 만들면 안 된다.**

### W3. "여기부터가 '됨'" → 뜻이 통하는 문구

- **무엇을**: 보드 상태 흐름 도식의 '완료' 박스 주석 `여기부터가 '됨'` 을 교체한다. 처음 보는 독자에게 뜻이 통하지 않는다.
- **권장 문구**: **`검증까지 끝남`**
  - *왜*: 바로 앞 '개발 완료' 박스의 주석이 **"아직 검증 전"**이다. 둘을 **검증이라는 같은 축의 전/후 대비**로 맞추면, 도식만 봐도 "개발 완료는 아직 안 끝난 것, 완료는 검증까지 끝난 것"이 즉시 읽힌다.
  - 대안(동급): `검증까지 끝나야 완료` / `여기 와야 완료로 집계`
- **수용 기준**:
  - [ ] `'됨'` 표현이 사라지고, 처음 보는 독자가 뜻을 알 수 있는 문구로 교체.
  - [ ] '개발 완료'(아직 검증 전) ↔ '완료' 주석이 **대비 쌍**으로 읽힌다.
  - [ ] 도식의 다른 요소·색·강조는 그대로.

## 제약

- **본문 서술·구조·수치는 이 문서에 적힌 것 외 무변경.**
- 정적 유지(새 라이브러리 0), 기존 토큰만, 다크/라이트 정상.
- 민감정보 0 유지.

## 검증 (개발자 규칙)

- **라벨 잘림 눈검증** — 데스크톱 + 좁은 화면(모바일 폭) 둘 다에서 타임라인 캡처해 확인. 세그먼트 폭 비율 유지 확인.
- 색 없이도 구간 식별 가능한지 확인(보조 인코딩 유지).
- `회귀` 잔존 위치가 **의도한 현상 서술만**인지 grep으로 확인.
- 도식 문구 교체 확인, aria-label도 함께 갱신됐는지.
- 빌드 green, 본문 리터럴 `**`·`<del>` 0, 민감정보 0.
- 완료 보고는 돌린 확인 + 결과로.
