# 콘텐츠 스타일 정책

- **문체**: 여정기(`repetition-to-ai-judgment-to-human`)만 존댓말(-습니다) — 독자에게 말 거는 종합 회고이기 때문. 나머지 개별 포스트는 평서체(-다)로 통일. 한 글 안에서 섞지 않는다.
- **표기**: "릴리스"(O) / "릴리즈"(X).
- **수치**: 실측값과 어림값을 구분해 표기한다("약 600건" vs "실측 615건"). 근거 없는 100%·전량류 표현 금지.
- **도구명**: Jira·Confluence·Figma·Google Sheets 등 도구 이름은 실명 표기(첫 등장 정식명, 이후 축약 가능). 사내 서버 주소·티켓 번호·고객사명·크리덴셜은 계속 비공개.
- **영어 용어**: 비개발 독자용 글에서는 첫 등장 시 우리말 병기 — 예: "상황판(wallboard)", "off-by-one(참조가 한 행씩 밀리는) 에러".

## 용어 정책 (2026-07-23 추가)

1. **병기 대상은 난이도가 아니라 "이 글의 논지에 필요한가"로 정한다.** 논지에 안 걸리는 말은 그냥 쉬운 우리말로 쓰고 괄호를 붙이지 않는다.
2. **최초 1회만 병기**, 이후 한 표기로 고정한다. 같은 글에서 두 번 병기하지 않는다.
3. **영문 유지 / 우리말 전환 목록**:
   - 영문 유지: `Regression`, `Code Freeze`, `catch-all`, `off-by-one`, `SSOT`, `Postmortem`, `contract`(계약 검증)
   - 우리말: `상황판`(월보드 ✗), `범위에서 뺀다`(de-scope 병기 가능), `모범사례`(best practice), `자체 운영`(셀프서브 ✗), `중간 단계`(hop ✗), `갭`(gap 영문 ✗)
4. **음차 금지** — 우리말로 자연스러운 말이 있으면 소리만 옮기지 않는다 (사례: 브리틀→"쉽게 깨진다", 리그레션→Regression, 월보드→상황판).
5. **풀이는 원 개념과 일치해야 한다** — 뜻이 어긋나면 병기가 오히려 해가 된다 (사례: 커넥션 풀은 '대기열'이 아니라 '미리 열어두고 돌려 쓰는 연결 묶음', ITIL은 '표준'이 아니라 '모범사례 체계').
