# 개발 핸드오프 — About 페이지 리디자인 (2026-07-23)

> ## 🗣 쉬운 요약
> - **무엇**: About 페이지를 **프로필 형식(LinkedIn 느낌)**으로 다시 만듭니다. 내용은 그대로, **구조와 배치만** 바꿉니다.
> - **왜**: 지금은 제목·불릿만 있는 평평한 산문이라 경력이 "나열"로 읽힙니다. LinkedIn이 정돈돼 보이는 건 화려해서가 아니라 **구조가 있어서**입니다 — 프로필 헤더, 기간 정렬, 회사 아래 역할 묶기, 카드 분할.
> - **그래서**: 반복 구조(경력·강점·원칙)를 데이터로 분리하고 카드·타임라인으로 렌더합니다. **블로그 톤은 그대로 유지**합니다.

> 🔴 **최우선 제약 — 내용 손실 0**: About은 채용담당자가 읽는 페이지다. 현재 문장·경력·수치·링크를 **하나도 빠뜨리지 않는다.** 재배치는 하되 삭제·요약은 하지 않는다(도입 산문의 미세 압축만 허용).

## 구현 방식

- 현재: `src/content/pages/about.md`(마크다운 산문) → `src/pages/about.astro`가 `app-prose`로 렌더.
- 변경: **`about.astro`에 데이터 배열 + 레이아웃**을 두고 구조를 렌더한다. `series/ai-teammate.astro`가 이미 같은 패턴(`SERIES_ORDER`·`ROLES` 배열 + 카드)이니 그 방식을 따른다.
- `about.md`는 **메타(title·description) + 도입 산문**만 남기거나, 전량 astro로 옮기고 메타만 유지 — **판단은 개발자 재량**. 다만 **콘텐츠의 단일 출처가 어디인지 명확**해야 한다(둘로 쪼개 흩어놓지 말 것).

## 📋 개발 핸드오프

### A1. 프로필 헤더 (상단)

- **무엇을**: 페이지 최상단을 프로필 헤더로. 지금은 이름이 본문 첫 문장에 묻혀 있다.
- **구성**:
  - **이니셜 아바타** — 원형, accent 계열 배경에 이니셜. *사진은 넣지 않는다*(블로그에 인물 사진이 없고 톤과 안 맞음). 저자가 원하면 나중에 사진으로 교체 가능한 구조로.
  - **이름**(크게) + **한 줄 타이틀** — 예: "10년차 QA 엔지니어 · QA 프로세스 설계와 AI 업무 자동화(AX)"
  - **요약 문단** — 현재 도입 2문단. 그대로 두되 헤더 아래 배치.
  - **연락 버튼** — GitHub · LinkedIn을 **상단으로 승격**(현재는 페이지 맨 아래 목록). 아이콘+라벨 버튼 형태.
  - **핵심 지표 칩 3~4개** — 예: `10년차` · `도메인 4개 전환` · `1인 QA` · `AI 팀원 3인`. 홈 성과 스트립과 **값이 겹치지 않게** About 성격으로.
- **수용 기준**:
  - [ ] 이름·타이틀·연락 버튼이 **첫 화면에서 바로** 보인다.
  - [ ] 지표 칩이 홈 스트립과 중복되지 않는다.
  - [ ] 사진 없음. 아바타는 이니셜.

### A2. 경력 타임라인 (핵심 — 여기서 인상이 갈린다)

- **무엇을**: 경력을 **타임라인/카드**로. 지금은 h3 + 불릿이라 회사 간 구분이 약하다.
- **구성**:
  - 항목마다 **회사 · 역할**(좌) ↔ **기간**(우측 정렬). 이 정렬 하나로 프로필 느낌이 크게 산다.
  - **넷마블 2건은 한 회사 아래 역할 2개로 묶는다** — Part Lead(2023.01~2024.04)와 Manager(2020.08~2023.01). LinkedIn의 회사-역할 계층과 같은 방식이고, 승진 이력이 자연히 드러난다.
  - 각 항목: 한 줄 컨텍스트 + 성과 불릿(현재 내용 그대로).
  - 좌측 세로선 + 노드(점) 또는 카드 스택 — **택1, 과하지 않게**.
- **수용 기준**:
  - [ ] 회사·역할·기간이 한눈에 구분된다(기간 우측 정렬).
  - [ ] **넷마블 2역할이 한 회사로 묶여** 있다.
  - [ ] 기존 성과 불릿과 **본문 내 링크가 전부 보존**된다(릴리스 트레인·자동화 인프라·관제 대시보드·AI 팀원 시리즈·여정기).

### A3. 강점 · 일하는 원칙 → 카드 그리드

- **무엇을**: 번호 목록(강점 4 / 원칙 4)을 **2열 카드 그리드**로. 각 카드는 굵은 제목 + muted 설명.
- **수용 기준**:
  - [ ] 강점 4개, 원칙 4개가 **각각 카드 그리드**로 렌더된다.
  - [ ] 모바일에서 1열로 접힌다.
  - [ ] 문구 무변경.

### A4. 마무리 인용 + Contact 정리

- **무엇을**: `> 반복은 AI에게, 판단은 사람에게.` 를 **강조 배너**로(현재는 일반 인용). 하단 Contact 목록은 헤더로 올렸으니 **중복이면 제거**, 남긴다면 형태를 달리한다.
- **수용 기준**: [ ] 인용이 시각적으로 마무리 역할을 한다. [ ] 연락처가 두 곳에 같은 형태로 중복되지 않는다.

## 디자인 제약 (블로그 톤 유지 — 중요)

- **기존 토큰만** 사용: `--accent`, `--muted`, `--border`, `--foreground`, `--muted-foreground`. **새 색 추가 금지.**
- 카드는 기존 `Card.astro`와 같은 결: `rounded-xl` + `border border-border` + `bg-muted/40`.
- 폰트는 Pretendard 그대로. **새 폰트·아이콘 세트·애니메이션 라이브러리 금지.**
- **금지**: 인물 사진, 그라데이션 배경, 글래스모피즘, 큰 히어로 이미지, 과한 그림자. 이 블로그의 미덕은 **절제**다.
- 다크/라이트 **양쪽 정상**. 정적 유지(런타임 JS 0).
- 반응형: 모바일에서 타임라인·그리드가 자연스럽게 1열로.
