> 소스 코드에서 화면 UI를 추출해 Figma 스토리보드로 자동 재구성하는 워크플로를 만들어, 기획 문서 공백을 대규모로 메웠다.

## 배경

운영 중인 키오스크·관리 플랫폼은 화면 수가 많고 기획서가 낙후되어 있었다. QA·기획·신규 입사자가 화면을 이해하려면 '지금 화면'을 기준으로 한 스토리보드가 필요했다.

## 무엇을 만들었나

- 소스에서 화면 구조·전환 흐름·문구키를 추출하고, Figma에 화면 + 스펙 패널을 쌍으로 자동 배치. 화면의 번호 배지와 기획 컴포넌트를 1:1 매핑.
- **3단 설명 규격:** 이름(코드명 금지, 평이한 우리말) · 설명(기획 톤) · 스펙줄(크기·문구키 등 개발 참고만 분리). 공통 헤더는 한 번 정의 후 참조(DRY).
- 키오스크 2종(입금·출금 대칭)과 관리 플랫폼의 전 화면 + 버튼 클릭 시 뜨는 팝업/모달까지 전수 역기획 완료.

## 산출물 샘플 — 스펙 패널은 이렇게 생겼다

화면 옆에 붙는 스펙 패널의 실제 구조 (도메인 각색 예시). 화면 위 번호 뱃지 ①②③…과 아래 항목이 1:1로 대응한다.

> **③ 시작하기 버튼**
>
> 화면 중앙의 큰 주황색 버튼. 손님이 이 버튼을 누르면 거래가 시작됩니다. 장비 점검 중이거나 준비가 안 됐을 때는 눌러도 반응하지 않고, 화면 상단에 점검 안내가 표시됩니다.
>
> 🛠 `520×160px · 문구키 btn_start · 이미지 btn_start_2.png`

- **1행 이름** — 코드명 금지, 평이한 우리말 ("btnStart" ❌ → "시작하기 버튼" ✅)
- **2행 설명** — 기획 톤. "무엇이 일어나는가"를 사용자 관점으로. 이 서술은 **코드에서 확인한 실제 흐름**이다 — 추측이나 옛 기획서 복사가 아니라.
- **3행 🛠 스펙줄** — 크기·문구키·리소스명 등 개발 참고만 작은 글씨로 격리. 비개발 독자는 이 줄을 건너뛰어도 된다.

화면 고유 모달은 6항목 표준으로 기술한다:

| 항목 | 예시 (관리자 인증 모달) |
| --- | --- |
| 트리거 | 관리자 메뉴 진입 시 자동 표시 |
| 입력 | 관리자 카드 태그 또는 비밀번호 |
| 내용 | 안내 문구 + 가상 키패드 |
| 액션 | 확인 / 취소 |
| 정책 | 3회 실패 시 초기 화면 복귀 |
| 결과 | 성공 → 관리자 메뉴, 취소 → 이전 화면 |

문서 하단에는 **디스크립션 TBD 표** — 코드만으로 알 수 없는 정책(금액 한도의 근거, 문구의 의도 등)을 번호별 빈칸으로 남기면, 담당자가 채우고 확정 반영하는 협업 루프가 돈다.

## 규모와 배치 규격

- 키오스크 2종 37화면 + 관리 플랫폼 30화면 + 모달 수십 종 전수. 화면마다 스펙 패널을 쌍으로, IA 그룹별 섹션 배치 — 좌표 규칙까지 문서화해 어느 세션이든 이어서 배치할 수 있게.
- **공통 모달 카탈로그** — 전 화면 공통(확인/이미지·텍스트·영상 뷰어/설정)은 1회 정의 후 참조, 화면 고유 모달은 그 화면 아래 태그로. 모달 상세는 6항목 표준(트리거·입력·내용·액션·정책·결과).
- **미러 화면 표준** — 대칭 제품은 clone 후 차이만 치환(치환 헬퍼 3종). 문서 하단에 디스크립션 TBD 표 — 번호별 빈칸을 담당자가 채우면 확정 반영하는 협업 루프.

## 시행착오

- **"소스에 없음" 오판** — 템플릿 폴더만 보고 화면이 없다고 판단했는데, 실제로는 다른 계층(JSP)에 있었다. 부재 결론은 전체 구조를 본 다음에만.
- **빌드 주의 누적 → 규칙화** — 카드 프레임 기본 클리핑이 모서리 배지를 자르고, 셀 프레임 사이즈 설정 누락 시 표가 찌그러지는 등 자잘한 함정을 만날 때마다 누적 규칙으로 문서화 — 다음 화면부터는 안 밟는다.
- **미러 잔재** — clone 복사 시 도메인 고유 동작 서술이 라벨 치환에 안 걸리고 살아남은 사고 → 타깃 소스 재검증 규칙.

## 결과

제품 3종의 기획 문서 공백이 '지금 화면' 기준 스토리보드로 메워졌다. 비개발 구성원도 리뷰할 수 있는 톤이라 기획·QA·신규 입사자의 공통 참조점이 되고, 플랫폼화 시 baseline으로 재사용된다.

> 동작 서술은 '소스 거동'이 아니라 '코드에서 확인한 실제 흐름'이어야 한다. 미러 화면을 복사할 때 도메인 고유 동작은 라벨 치환으로 안 걸리므로, 항상 타깃 소스로 재검증했다.
