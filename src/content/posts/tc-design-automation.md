> Figma 시안을 분석하여 Google Sheets에 TC를 자동 생성하는 핵심 스킬.

## 무엇을 만들었나

TC 설계 표준(363줄 규격 문서)을 정의하고, 이 규격에 따라 AI가 Figma 시안을 분석해 TC를 Google Sheets에 직접 작성하는 워크플로우를 구축했다.

- 하이브리드 분석: 메타데이터 우선 + 선별적 스크린샷 보완
- Full TC + Smoke TC 2시트 구조 자동 생성
- Summary 수식, Priority 조건부 서식, 드롭다운, 컬럼 너비, Freeze Pane(틀 고정)까지 자동 적용
- Hidden 노드 자동 제외, 비-UI 프레임(메모·설명) 필터링
- 프로젝트 무관하게 md 규격만 있으면 동일 품질의 TC가 생성됨

## 어떤 문제를 겪었나

### 1. 서식이 매번 빠지거나 깨졌다

초기에는 TC 데이터만 넣고 서식을 빼먹거나, 적용 순서가 잘못되어 테두리가 덮어쓰기 되는 문제가 반복되었다.

- **해결:** '서식 적용 체크리스트 9단계'를 규격에 명시. 특히 Border는 반드시 마지막에 적용하도록 순서를 고정했다.

### 2. Priority 조건부 서식 누락

P0~P3 글자색이 적용되지 않는 경우가 반복되었다. 체크리스트에 별도 항목으로 추가하여 누락을 방지했다.

### 3. 드롭다운 칩 색상의 API 한계

Google Sheets API로는 드롭다운 칩(pill)의 색상을 설정할 수 없다. 조건부 서식으로 대체하면 칩 스타일이 깨진다. 결론: 드롭다운만 API로 설정, 색상은 수동으로 남기는 것이 최선이었다.

### 4. 메타데이터만으로는 부족했다

노드명이 title, description처럼 일반적인 이름인 경우 실제 UI 텍스트를 알 수 없었다. 핵심 화면만 선별해 스크린샷으로 보완하는 하이브리드 방식으로 해결했다.

## 핵심 교훈

> 규격 문서의 품질이 곧 산출물의 품질이다. 서식 규칙을 '체크리스트'로 단계화하면 AI가 빠뜨리지 않는다.

- TC 작성 규칙도 '주니어 QA가 바로 수행 가능한 수준'을 기준으로 명시해야 한다. 추상적 표현을 쓰면 AI도 추상적으로 쓴다.
- API 한계를 빨리 파악하고 '수동 영역'으로 명확히 분리하면 시간 낭비가 줄어든다.
